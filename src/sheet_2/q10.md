### Can you implement a method for `Vec<i32>` directly?

No, you **cannot** implement your own inherent methods directly on `Vec<i32>` in your crate.

Example of what **is NOT allowed**:

```rust
impl Vec<i32> {
    fn sum_all(&self) -> i32 {
        self.iter().sum()
    }
}
```

This fails because `Vec` is defined in the **Rust standard library**, and Rust does not allow you to add inherent implementations to types you don't own.

---

# Why? — The Orphan Rule

In **Rust**, the **Orphan Rule** states:

> You can implement a trait only if **either the trait or the type is defined in your crate**.

This prevents conflicts between crates.
Imagine two crates both doing:

```rust
impl SomeTrait for Vec<i32>
```

Rust wouldn't know which implementation to use.

---

# Workaround: The Newtype Pattern

The common solution is the **Newtype Pattern**, where you wrap the external type inside your own struct.

### Step 1: Create a wrapper

```rust
struct MyVec(Vec<i32>);
```

Now `MyVec` is **your own type**, so you can implement methods on it.

---

### Step 2: Implement methods

```rust
impl MyVec {
    fn sum_all(&self) -> i32 {
        self.0.iter().sum()
    }
}
```

---

### Step 3: Example usage

```rust
fn main() {
    let v = MyVec(vec![1, 2, 3, 4]);

    println!("{}", v.sum_all()); // 10
}
```

Here:

```
self.0
```

accesses the inner `Vec<i32>`.

---

# Optional: Make it behave like Vec

You can also implement **Deref** so it acts like a `Vec`.

```rust
use std::ops::Deref;

impl Deref for MyVec {
    type Target = Vec<i32>;

    fn deref(&self) -> &Self::Target {
        &self.0
    }
}
```

Now you can do:

```rust
let v = MyVec(vec![1,2,3]);
println!("{}", v.len());
```
